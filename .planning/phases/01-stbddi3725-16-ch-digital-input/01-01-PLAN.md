<objective>
Lock the channel-to-bit mapping convention for the entire STB milestone and ship the foundational `IO16LedBlockPainter` that all 16-channel STB modules (DDI3725 here, DDO3705 in Phase 2) render through. This plan establishes:

1. A module-level constant `kSTBChannelBitOrder` (default `STBBitOrder.lsbFirst` — bit 0 → channel 1) that any future module imports rather than re-deriving.
2. A pure `bitmaskToLedStates(int raw, {int? forceValues})` helper that converts a uint16 bitmask (+ optional int8[16] force-values per D-XX) into `List<IOState>` of length 16 with force collapsing raw (forced channels ignore raw bit — matches `BeckhoffEL1008._ledStates`).
3. `IO16LedBlockPainter extends BaseLedBlockPainter` with a 2×8 column-major LED layout (channels 1–8 LEFT column top→bottom, 9–16 RIGHT column top→bottom — per `.planning/research/photos/DDI3725_front_clean.png`).
4. A bit-mapping unit test that LOCKS the convention. If Schneider's actual convention is MSB-first the constant flips + the three assertions update; nothing else moves.

Purpose: bit-ordering and force-collapse semantics are the single highest-risk technical decisions in this milestone (PITFALLS M-02, M-04). Encoding them as a public constant + a tested pure function quarantines the risk: the painter, widget, dialog, goldens, and the entire DDO3705 phase all consume `bitmaskToLedStates` rather than re-implementing bit math.

Output: `lib/painter/advantys_stb/io16.dart` (~120 LoC) + the bit-mapping test group in `test/page_creator/assets/advantys_stb_test.dart` (~80 LoC). After this plan, Plan 02 can build the DDI3725 body painter and widget without touching bit math.
</objective>

The 8-channel sibling at io8.dart:432-453 is the layout precedent (row-major 2×4):
```dart
@override
void _drawLeds(Canvas canvas, Size size) {
  final pad = size.width * 0.05;
  const cols = 2, rows = 4;
  final cellW = (size.width - pad * (cols + 1)) / cols;
  final cellH = (size.height - pad * (rows + 1)) / rows;
  final borderPaint = Paint()..color = Colors.grey.shade700..style = PaintingStyle.stroke
                              ..strokeWidth = size.width * 0.03;
  for (int i = 0; i < 8; i++) {
    int r = i ~/ cols, c = i % cols;
    double cx = pad + c * (cellW + pad);
    double cy = pad + r * (cellH + pad);
    final cellRect = Rect.fromLTWH(cx, cy, cellW, cellH);
    _drawLed(canvas, cellRect, ledStates[i], borderPaint);
  }
}
```

**For IO16 the layout MUST be column-major 2×8** — that is, ledStates[0..7] fill the left column top→bottom, ledStates[8..15] fill the right column top→bottom. The loop maps `i → (col: i ~/ 8, row: i % 8)` (NOT `r = i ~/ cols, c = i % cols`). See UI-SPEC §Layout.
