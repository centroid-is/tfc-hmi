`Widget _statusDialog(BuildContext context, StateMan stateMan)` (the EL1008 method):
- Returns `AlertDialog` with `title: Text(config.nameOrId)`.
- `content:` is a `StreamBuilder<Map<String, DynamicValue>>` whose stream is a fresh `_combinedStream` over all six keys (`raw`, `processed`, `force`, `descriptions`, `on_filters`, `off_filters`).
- Builder body: extracts `rawStates: List<bool>(8)` from `(map["raw"]!.asInt & (1 << i)) != 0`.
- For each row `i` in `0,2,4,6` (EL1008 has 4 rows × 2 cols):
  - `RowIOView(leftRaw: rawStates?[i], rightRaw: rawStates?[i+1], leftSelected: map["force"]?[i].asInt ?? 0, rightSelected: ..., animationValue: animation, leftOnChanged: async => stateMan.write(config.forceValuesKey!, mutated_map), rightOnChanged: ..., leftDescription: map["descriptions"]?[i].asString, rightDescription: ..., leftFilterEdit: FilterEdit(...), rightFilterEdit: FilterEdit(...))`

For DDI3725 (16 channels, 8 rows × 2 cols):
- Extract `rawStates: List<bool>(16)`.
- Pair channel `i` (left) with channel `i+8` (right) — UI-SPEC §Detail Dialog locks this. Row r=0..7: left index = r, right index = r+8.
- Loop: `for (int r = 0; r < 8; r++)` ... `leftRaw: rawStates?[r], rightRaw: rawStates?[r+8], leftSelected: map["force"]?[r].asInt ?? 0, rightSelected: map["force"]?[r+8].asInt ?? 0, ...`.
- Mutating force: `map["force"]![r].value = newValue; await stateMan.write(config.forceValuesKey!, map["force"]!);` (same pattern as EL1008 at beckhoff.dart:1397-1405).

`RowIOView` constructor signature (from beckhoff.dart:1597+ — confirm exact named params during implementation):
```dart
RowIOView({
  required bool leftRaw, required bool rightRaw,
  bool? leftProcessed, bool? rightProcessed,
  required int leftSelected, required int rightSelected,
  required Animation animationValue,
  required Function(int) leftOnChanged, required Function(int) rightOnChanged,
  String? leftDescription, String? rightDescription,
  FilterEdit? leftFilterEdit, FilterEdit? rightFilterEdit,
})
```

`FilterEdit` (beckhoff.dart:1488-1545): two side-by-side `TextFormField`s ("On filter ms", "Off filter ms"). Bound to `onFiltersKey` / `offFiltersKey` int16 arrays. Mutation: `map["on_filters"]![r].value = newValue; await stateMan.write(config.onFiltersKey!, map["on_filters"]!);`.

Import strategy: at the top of `advantys_stb.dart` add `import 'beckhoff.dart' show RowIOView, FilterEdit;`. Per ARCHITECTURE §9.3, do NOT duplicate `RowIOView` / `FilterEdit` — they are public widgets in `beckhoff.dart` and re-using them is the correct cross-vendor pattern (the widgets are vendor-neutral despite living in `beckhoff.dart`).

`_combinedStream` (already in `advantys_stb.dart` from Plan 02): re-used. Dialog stream subscribes to all FIVE keys (raw, force, on_filters, off_filters, descriptions) — distinct from the body stream which only subscribes to raw + force.
