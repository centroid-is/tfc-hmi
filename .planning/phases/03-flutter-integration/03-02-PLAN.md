       **Replace the UpgradeAlert + onUpdate block (around line 116-125):**

       Change FROM:
       ```dart
       runApp(ProviderScope(
           child: UpgradeAlert(
         child: MyApp(locationBuilder: locationBuilder),
         upgrader: upgrader,
         onUpdate: () {
           stderr.writeln("Updating software from store");
           UpgraderWindowsStore.installUpdate();
           return false;
         },
       )));
       ```

       Change TO:
       ```dart
       runApp(ProviderScope(
         child: UpgradeAlert(
           upgrader: upgrader,
           onUpdate: () {
             final targetVersion =
                 upgrader.currentVersionInfo?.appStoreVersion?.toString() ?? '';
             unawaited(
               ManagerLauncher.launchForUpdate(
                 version: targetVersion,
                 flutterPid: pid,
               ).then((_) => exit(0)),
             );
             return false; // Prevent upgrader from calling sendUserToAppStore()
           },
           child: MyApp(locationBuilder: locationBuilder),
         ),
       ));
       ```

       CRITICAL details:
       - `onUpdate` MUST return `false` — returning true causes upgrader to call `sendUserToAppStore()` which no longer has a valid store URL
       - `pid` is the top-level `dart:io` getter for the current process PID
       - `unawaited()` wraps the future to avoid lint warnings about unhandled futures
       - `exit(0)` after launch ensures Flutter app exits so the manager can install the update
       - The `child:` parameter position is moved to the end per Dart convention (named parameter after positional)
