<objective>
Fix the FC15 Write Multiple Coils quantity bug in the modbus_client base library using TDD.

Purpose: `getMultipleWriteRequest` at `modbus_element.dart:109` uses `bytes.length ~/ 2` for the PDU quantity field -- correct for FC16 (registers, 2 bytes each) but wrong for FC15 (coils, 1 bit each packed into bytes). The quantity field must equal the number of coils, which cannot be recovered from the packed byte count. An explicit `quantity` parameter is needed.

Output: Local `packages/modbus_client/` fork with the one-line fix, comprehensive PDU-level tests, and `modbus_client_tcp` dependency chain updated to use the local fork.
</objective>

FC15 PDU layout (request):
```
[0]     = 0x0F          (function code FC15)
[1][2]  = address       (uint16 big-endian)
[3][4]  = quantity      (uint16 big-endian, NUMBER OF COILS -- THE BUG)
[5]     = byte count    (ceil(quantity / 8))
[6..N]  = packed bits   (coil values, LSB first per byte)
```
