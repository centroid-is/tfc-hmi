<objective>
Add automatic batch coalescing to ModbusClientWrapper's poll system: contiguous same-type registers in the same poll group are merged into single ModbusElementsGroup batch reads, with gap handling and automatic splitting for oversized batches.

Purpose: Batch reads dramatically reduce TCP round-trips. Reading 10 registers in one request vs 10 separate requests eliminates 9 round-trips (~40ms each = 360ms saved per poll cycle). This is standard SCADA practice and critical for HMI responsiveness at fast poll rates.

Output: _buildCoalescedGroups algorithm, updated _onPollTick to use batch reads, dirty flag for recalculation, and full TDD test coverage.
</objective>

    1. Add constants for gap thresholds:
       ```dart
       static const _registerGapThreshold = 10;
       static const _coilGapThreshold = 100;
       ```

    2. Add `List<ModbusElementsGroup> _cachedGroups = [];` field to _PollGroup.

    3. Implement `List<ModbusElementsGroup> _buildCoalescedGroups(List<_RegisterSubscription> subs)`:
       - If subs is empty, return []
       - Group subscriptions by element type (using `sub.element.type`)
       - For each type group:
         a. Sort by address (`sub.element.address`)
         b. Determine gap threshold: `type.isRegister ? _registerGapThreshold : _coilGapThreshold`
         c. Determine max range: `type.isRegister ? ModbusElementsGroup.maxRegistersRange : ModbusElementsGroup.maxCoilsRange`
         d. Walk through sorted elements, starting a new batch when:
            - Gap between previous element's end address and current element's start address exceeds threshold
            - Adding current element would make the batch range exceed max range
         e. For each batch, create `ModbusElementsGroup(batch.map((s) => s.element))`
       - Return all groups

       **Critical: Element end address calculation:**
       - For registers: `element.address + element.byteCount ~/ 2` (uint16=1, int32=2, float64=4)
       - For bits: `element.address + 1`

       **Critical: Batch range calculation before adding:**
       - `candidateEnd = currentElement.address + elementSize`
       - `batchRange = candidateEnd - batch.first.element.address`
       - If `batchRange > maxRange`, start new batch

    4. Update `_onPollTick(_PollGroup group)` to use coalesced groups:
       - If `group._dirty`, rebuild: `group._cachedGroups = _buildCoalescedGroups(group._subscriptions); group._dirty = false;`
       - Iterate `group._cachedGroups` instead of `group._subscriptions`
       - For each `ModbusElementsGroup elemGroup`:
         a. `final request = elemGroup.getReadRequest(responseTimeout: group.responseTimeout);`
         b. `final result = await _client!.send(request);`
         c. If `result == ModbusResponseCode.requestSucceed`:
            - Values are already parsed into each element by the library's `internalSetElementData`
            - Iterate ONLY subscriptions whose elements are in this group and pipe `sub.element.value` to `sub.value$.add()`
         d. If failure, log warning and continue to next group
       - After iterating groups, also pipe values for any subscriptions NOT in any group (edge case: empty groups)

       **Critical: Matching subscriptions to groups for value piping:**
       After `send()` succeeds, the library has already set each element's `.value`. Iterate ALL subscriptions in the poll group and pipe their element's current value:
       ```dart
       for (final sub in group._subscriptions) {
         if (!sub.value$.isClosed) {
           sub.value$.add(sub.element.value);
         }
       }
       ```
       This works because elements are shared references -- the group read populated the exact same element objects that subscriptions hold.

       Actually, wait -- this is simpler. After ALL groups in a poll tick have been read, pipe ALL subscription values once. This avoids matching subscriptions to groups entirely.

    6. Handle the edge case where a single element read would be needed (only one subscription in the group) -- ModbusElementsGroup works fine with a single element, so this is handled automatically.
