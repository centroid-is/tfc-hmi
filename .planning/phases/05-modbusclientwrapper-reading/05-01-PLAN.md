<objective>
Extend ModbusClientWrapper with poll-based reading of all four Modbus register types (coils, discrete inputs, holding registers, input registers) with configurable data types and named poll groups.

Purpose: This is the core read infrastructure for the Modbus integration. Without it, no Modbus values can flow into the HMI. Poll groups with connection-lifecycle-tied timers ensure registers are read at configured intervals while the device is connected.

Output: ModbusClientWrapper extended with subscribe/read/unsubscribe API, ModbusRegisterSpec config class, ModbusDataType enum, named poll groups with Timer.periodic, and full TDD test coverage.
</objective>
