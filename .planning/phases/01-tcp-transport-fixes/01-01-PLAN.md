    **TCPFIX-01 (frame length):**
    Change line 304 from:
    ```dart
    if (_resDataLen != null && _data.length >= _resDataLen!) {
    ```
    to:
    ```dart
    if (_resDataLen != null && _data.length >= _resDataLen! + 6) {
    ```
    This accounts for the 6-byte MBAP header prefix (transaction ID 2 + protocol ID 2 + length field 2) that is NOT included in the length field value.

    **TCPFIX-03 (length validation):**
    After line 301 (`_resDataLen = resView.getUint16(4);`), add validation:
    ```dart
    if (_resDataLen! < 1 || _resDataLen! > 254) {
      ModbusAppLogger.warning(
          "Invalid MBAP length field", "$_resDataLen not in range 1-254");
      _timeout.complete();
      request.setResponseCode(ModbusResponseCode.requestRxFailed);
      return;
    }
    ```
    Upper bound is 254 per Modbus spec (1 unit ID + max 253 PDU bytes).

    **TCPFIX-04 (TCP_NODELAY):**
    In the `connect()` method, add after `_socket = await Socket.connect(...)`:
    ```dart
    _socket!.setOption(SocketOption.tcpNoDelay, true);
    ```
    Place this BEFORE `_enableKeepAlive(_socket!)` so both socket options are set early.

    **TCPFIX-05 (keepalive values):**
    1. Rename `keepAliveInterval` constructor parameter to `keepAliveInterval` (keep name) BUT add a new `keepAliveIdle` parameter with a DIFFERENT default:

    ModbusClientTcp(this.serverAddress, {

      this.keepAliveIdle = const Duration(seconds: 5),

      // ...
    });
