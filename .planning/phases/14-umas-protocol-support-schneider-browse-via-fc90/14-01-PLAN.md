<objective>
Implement the UMAS protocol client that communicates with Schneider PLCs via FC90 (function code 90) over existing Modbus TCP transport.

Purpose: This is the protocol backbone that enables browsing Schneider PLC variables by name instead of requiring manual register address entry. UMAS rides on top of Modbus TCP using FC90, with little-endian payloads containing sub-function codes for init, PLC identification, and data dictionary reading.

Output: UmasClient class that can init communication, read the PLC data dictionary (variable names + data types), and build a hierarchical variable tree. All communication goes through the existing ModbusClientTcp.send() pipeline. The MBAP length validation is relaxed for FC90 to allow large UMAS data dictionary responses (500-5000 bytes).
</objective>

UMAS Frame Structure (from research):
```
Request: FC=0x5A, PairingKey(1 byte), SubFunction(1 byte), Payload(N bytes)
Response: FC=0x5A, PairingKey(1 byte), Status(0xFE=success/0xFD=error), SubFunction(1 byte on success), Payload(N bytes)
UMAS payloads are LITTLE-ENDIAN (unlike standard Modbus big-endian)
```

However, UMAS (FC90/0x5A) data dictionary responses routinely exceed this -- real PLCs send 500-5000 byte responses for 0x26/0xDD02 and 0x26/0xDD03 sub-functions. Without this fix, ALL UMAS data dictionary responses will be silently discarded.

    There are TWO places that enforce the 254-byte limit:

    1. In `_processIncomingBuffer()` (around line 218):
       ```dart
       if (lengthField < 1 || lengthField > 254) {
       ```
       Change to exempt FC90 by peeking at the function code byte (byte index 7 in the MBAP frame, which is unitId(1) + FC(1) at position 7):
       ```dart
       // Standard Modbus limits MBAP length to 1-254, but UMAS (FC90/0x5A)
       // responses can be much larger (data dictionary 500-5000 bytes).
       final functionCode = _rxBuffer.length > 7 ? _rxBuffer[7] : 0;
       final maxLength = (functionCode == 0x5A) ? 65535 : 254;
       if (lengthField < 1 || lengthField > maxLength) {
       ```

    2. In `_TcpResponse.addResponseData()` (around line 396):
       ```dart
       if (_resDataLen! < 1 || _resDataLen! > 254) {
       ```

       ```dart
       final maxLen = (request.functionCode.code == 0x5A) ? 65535 : 254;
       ```
