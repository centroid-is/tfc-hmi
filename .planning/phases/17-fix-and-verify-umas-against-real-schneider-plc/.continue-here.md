<context>
Root cause: ServerConfigBody is a composite widget with 4 sections. Each section independently loads config via async _loadConfig() showing CircularProgressIndicator. Flutter's pumpAndSettle waits for ALL animations to stop, but CircularProgressIndicator is an indeterminate animation that never stops. On fast local macOS, the Futures resolve so quickly that _isLoading flips to false before pumpAndSettle even checks. On slower CI runners (Ubuntu/Windows), the animation gets a chance to render and blocks pumpAndSettle forever.

Best fix approach: Either (a) replace every pumpAndSettle() in test files with pump(Duration) calls, or (b) better — make buildTestableServerConfig render only _ModbusServersSection instead of full ServerConfigBody, avoiding the other sections' CircularProgressIndicators entirely.
