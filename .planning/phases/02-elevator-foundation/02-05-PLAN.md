<objective>
Wire the Elevator config dialog and register `ElevatorConfig` with `AssetRegistry`. This plan closes Phase 2 — operators can place an Elevator from the palette, saved pages with elevators round-trip, and saved pages without elevators still load cleanly.

Purpose: Plan 02-04 returned a placeholder `AlertDialog(title: Text('Configure Elevator'))` from `configure(BuildContext)`. Plan 02-05 replaces that with the real editor (`_ElevatorConfigEditor` — a `StatefulWidget` mirroring `_SensorConfigEditor` from `sensor.dart`). The dialog binds:
- `positionKey` (KeyField — ELEV-04 user-facing surface),
- `tweenDurationMs` (numeric TextFormField — surfaces the CONTEXT specifics §Tween duration default of 250 and lets the operator tune per-instance),
- `coordinates` + `size` (CoordinatesField with angle slider + SizeField — same as Sensor).

The locked `ElevatorChildEntry` schema from Plan 02-02 stays empty in Phase 2; the editor surfaces a read-only "Children: 0 (managed in Phase 3)" placeholder so the visual layout is fixed and Phase 3 only has to insert the list-management UI without restructuring the dialog.

Registry registration adds `ElevatorConfig` to both factory maps so the asset appears in the palette (ELEV-16) and saved pages round-trip (ELEV-17). Pitfall 5 (register-in-both) is closed by a single TDD-style registry round-trip + back-compat group in `elevator_config_test.dart`.

This is `type: execute` (mostly) with one TDD task — the registry round-trip + back-compat group. The dialog itself is plain Material UI glue (per CONTEXT TDD-mode policy: "config-dialog UI glue → standard plan"); the registry round-trip earns a RED→GREEN cycle because Pitfall 5 is the canonical failure mode for registry plumbing and the test is cheap to write.

            TextEditingController(text: widget.config.tweenDurationMs.toString());

        super.dispose();
      }

      @override
      Widget build(BuildContext context) {
        final config = widget.config;

        return Container(

                  ],
                  decoration: const InputDecoration(
                    labelText: 'Tween Duration (ms)',
                    hintText: '250',
                    helperText: 'Smoothing for platform position changes',
                  ),
                  onChanged: (v) {
                    final parsed = int.tryParse(v);
                    if (parsed != null && parsed >= 0) {
                      setState(() => config.tweenDurationMs = parsed);
                    }
                    // Empty / non-numeric input: leave config.tweenDurationMs
                    // unchanged so the runtime keeps the last valid value.

                // -- Size --

                  initialValue: config.size,
                  onChanged: (v) => setState(() => config.size = v),

                //    same as Sensor / Conveyor) --

                  onChanged: (c) => setState(() => config.coordinates = c),
                  enableAngle: true,

                const SizedBox(height: 16),
                const Divider(),

                //    add/edit/delete UI per the conveyor.dart precedent) --
