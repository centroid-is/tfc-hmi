# Plan 04-02: Golden-Review Fixes (Polish)

## Objective

Fix three issues caught by a user-driven visual review of all 14 golden PNGs. The existing test suite is regression-only — goldens verify "still looks the same", not "looks right". Each fix is a small, surgical change with a precise unit/widget test and a golden update.

This plan does NOT add new features. It corrects three pieces of polish that drifted from CONTEXT and/or from the visual contract:

1. **Fix 1 — ELEV-02 colour drift:** ElevatorPainter's default active colour is Material blue 700 (`#1976D2`) but 02-CONTEXT locked rails+platform as **neutral grey**. Change to `Color(0xFF757575)` (Colors.grey.shade600).
2. **Fix 2 — Children disappear at progress=100%:** `_buildPositionedChild` computes `top = platformOffsetTop(...) - childH`, which goes negative at high progress and pushes children out of the bbox. Clamp `top` to `>= 0` so children remain visible.
3. **Fix 3 — SENS-13 label invisible:** Sensor painters paint labels in `inactiveColor` (a light grey by default), which blends with the panel background. Labels are operator-facing tags ("PE-101A") and MUST be visible. Change to `Colors.black87`.
