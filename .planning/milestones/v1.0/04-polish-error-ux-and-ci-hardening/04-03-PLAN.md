# Plan 04-03: Editor Z-Order Reordering

## Objective

Add z-order reordering controls to the elevator's child list in `_ElevatorConfigEditor`. Today, when operators add multiple sensors/conveyors to the same elevator and they overlap on the platform, paint order is locked to add-order — there is no way to bring a critical sensor to the front (or push a less-important conveyor behind). This plan adds up/down arrow buttons to each child row that swap z-order in the underlying `config.children` list.

The editor list is rendered in **REVERSED** order (Photoshop / Figma convention) — topmost-paint at the top of the editor list, so:
- Up arrow ("Move forward") raises z — moves later in `config.children` (toward higher index = paints later = on top)
- Down arrow ("Move backward") lowers z — moves earlier in `config.children` (toward lower index = paints first = behind)

The Stack composition in `build()` already iterates `config.children` in order (index 0 painted first = lowest z). No change is needed there — reordering the list naturally re-orders paint order.

      ```dart
      /// Swaps `entry` with its `delta`-neighbour in `widget.config.children`.
      /// `delta = +1` raises z (moves later in the list = paints on top).
      /// `delta = -1` lowers z (moves earlier in the list = paints behind).
      ///
      /// Looks up the index by `entry.id` (not by widget order) so the
      /// handler is robust against display reversal — the editor displays
      /// `children.reversed`, but mutations are always on the underlying
      /// list. No-op if the entry is missing or the swap target is out of
      /// bounds (defensive — UI buttons disable at the boundary, but the
      /// handler is the safety net).
      void _onReorderChildPressed(ElevatorChildEntry entry, int delta) {
        final list = widget.config.children;
        final i = list.indexWhere((e) => e.id == entry.id);
        if (i < 0) return;
        final j = i + delta;
        if (j < 0 || j >= list.length) return;
        setState(() {
          final tmp = list[i];
          list[i] = list[j];
          list[j] = tmp;
        });
      }
      ```
