<completed_work>

- CENTROID_STDOUT=1 env var → opens console window with all output
- CENTROID_LOG_FILE=<path> env var → writes all output to a log file
- C++ runner (main.cpp, utils.cpp, utils.h): freopen_s + SetStdHandle + sync_with_stdio captures native/open62541 output
- Dart side (main.dart): runZoned with custom print handler captures Flutter print() to same log file (needed because Flutter print() does NOT route through C stdout in MSIX)
- MSIX quirk documented: _fileno(stdout) starts as -2 in MSIX GUI apps, must _dup2 to fd 1/2 explicitly
- Two commits: feat + rename CENTROID_DEBUG → CENTROID_STDOUT
</completed_work>

<decisions_made>

- Used CENTROID_STDOUT instead of CENTROID_DEBUG (more explicit about what it does)
- Used runZoned with custom print handler on Dart side because Flutter's print() does NOT go through C stdout in MSIX (confirmed through extensive testing)
- File redirect uses freopen_s (not CreateFileA) because it properly associates with CRT FILE*
- Must NOT call FlutterDesktopResyncOutputStreams() for file redirect — it tries CONOUT$ which destroys the file
- For MSIX writable paths, use LocalCache: %LOCALAPPDATA%\Packages\<PFN>\LocalCache\
</decisions_made>
